# Power BI Dashboard Build Guide

# Retail Revenue Intelligence Dashboard

This guide explains how to manually build the final Power BI dashboard from the prepared CSV files. Do not generate the `.pbix` automatically.

## 1. Open Power BI Desktop

Open Power BI Desktop and create a new blank report.

## 2. Import The Dashboard CSV Files

Use **Home > Get Data > Text/CSV** and import every file from:

```text
dashboard/data/
```

Import these files:

- `dashboard_kpi_summary.csv`
- `dashboard_monthly_sales.csv`
- `dashboard_weekly_revenue.csv`
- `dashboard_country_performance.csv`
- `dashboard_product_performance.csv`
- `dashboard_customer_segments.csv`
- `dashboard_segment_summary.csv`
- `dashboard_forecast_summary.csv`
- `dashboard_weekly_actual_vs_forecast.csv`
- `dashboard_forecast_model_comparison.csv`
- `dashboard_future_weekly_forecast.csv`
- `dashboard_future_monthly_forecast.csv`

## 3. Apply The Theme

Go to **View > Themes > Browse for themes** and select:

```text
dashboard/powerbi/powerbi_theme.json
```

## 4. Check Data Types

In Power Query or Model view:

- Set revenue, quantity, AOV, percentages, and scores to Decimal Number or Whole Number as appropriate
- Set `week_start_date` to Date
- Keep `invoice_year_month` and `forecast_month` as text labels unless you create date helper columns
- Set customer and product IDs to Text

## 5. Create Relationships If Needed

Recommended simple relationships:

- `dashboard_segment_summary[customer_segment]` to `dashboard_customer_segments[customer_segment]`
- `dashboard_country_performance[country]` to `dashboard_customer_segments[main_country]`

Use single-direction filtering from summary tables to detail tables.

## 6. Create The DAX Measures

Open `dashboard/powerbi/powerbi_dax_measures.md`.

Create a dedicated table named `Measures` and add the documented measures.

## 7. Build Page 1: Executive Overview

Add:

- KPI cards for revenue, orders, customers, AOV, repeat customer rate, and cancellation rate
- Monthly revenue line chart
- Revenue by country chart
- Top products by revenue chart
- Short executive insight text box

## 8. Build Page 2: Sales Performance

Add:

- Monthly revenue trend
- Weekly revenue trend
- Average order value over time
- Revenue by country
- Orders by country
- Country, month, and product slicers

## 9. Build Page 3: Customer Segmentation

Add:

- Customer count by segment
- Revenue by segment
- Segment summary table
- High-value customers table sorted by monetary value
- At-risk customers table filtered to `At Risk` and `Cannot Lose Them`
- Segment recommendation cards

## 10. Build Page 4: Product & Country Performance

Add:

- Top products by revenue
- Top products by quantity
- Country revenue ranking
- Average order value by country
- Product performance table
- Country performance table

## 11. Build Page 5: Forecasting & Recommendations

Add:

- Weekly actual revenue trend
- Actual vs forecast weekly revenue line chart using `dashboard_weekly_actual_vs_forecast.csv`
- Future weekly forecast
- Future monthly forecast
- Forecast model comparison table using `dashboard_forecast_model_comparison.csv`
- Forecast direction card
- Forecast recommendation cards
- Forecast limitation text:

```text
Forecasts are directional estimates because the available monthly history is limited.
```

## 12. Add Slicers

Use slicers sparingly:

- Country
- Month
- Product
- Customer segment
- Forecast model

Keep slicers aligned at the top or left side of each page.

## 13. Check Formatting

Use:

- Consistent card sizes
- Navy headings
- Blue, purple, and cyan accents
- Light backgrounds
- Minimal gridlines
- Clear chart titles
- Currency and percentage formatting

## 14. Export Screenshots

After building the dashboard, export screenshots manually to:

```text
dashboard/dashboard_screenshots/
```

## 15. Save The Power BI File

Save the final manual Power BI file as:

```text
dashboard/Retail_Revenue_Intelligence_Dashboard.pbix
```

This file is intentionally not generated by the Python project.
